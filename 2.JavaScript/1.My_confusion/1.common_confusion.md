## üß† 1. `==` vs `===`

| Operator | Description                                        | Example             |
| -------- | -------------------------------------------------- | ------------------- |
| `==`     | *Loose equality* ‚Üí converts types before comparing | `'5' == 5` ‚úÖ true   |
| `===`    | *Strict equality* ‚Üí compares type + value          | `'5' === 5` ‚ùå false |

‚úÖ **Always prefer `===`** for predictable comparisons.

## üîÑ 2. `var` vs `let` vs `const`

| Keyword | Scope           | Reassignment | Redeclaration | Notes                           |
| ------- | --------------- | ------------ | ------------- | ------------------------------- |
| `var`   | Function-scoped | ‚úÖ Yes        | ‚úÖ Yes         | Old-style, avoid if possible    |
| `let`   | Block-scoped    | ‚úÖ Yes        | ‚ùå No          | Use for mutable variables       |
| `const` | Block-scoped    | ‚ùå No         | ‚ùå No          | Use for constants or immutables |

```js
{
  let a = 10;
  var b = 20;
}
console.log(b); // ‚úÖ works
console.log(a); // ‚ùå ReferenceError
```

## ‚öôÔ∏è 3. `null` vs `undefined`

| Value       | Meaning                            | Example                               |
| ----------- | ---------------------------------- | ------------------------------------- |
| `undefined` | Variable declared but not assigned | `let a; console.log(a)` ‚Üí `undefined` |
| `null`      | Intentional absence of value       | `let a = null;` means ‚Äúno value yet‚Äù  |


## üß© 4. `function` vs `arrow function`

| Feature            | `function`         | `=>` arrow function               |
| ------------------ | ------------------ | --------------------------------- |
| `this` binding     | Has its own `this` | Inherits `this` from parent scope |
| `arguments` object | Available          | ‚ùå Not available                   |
| Hoisting           | ‚úÖ Yes              | ‚ùå No                              |

```js

```

## üóÉÔ∏è 5. `object`, `array`, and `Map`

| Type        | Structure                      | Access         | Example               |
| ----------- | ------------------------------ | -------------- | --------------------- |
| Object `{}` | Key-value pairs                | `obj.key`      | `{ name: "Alice" }`   |
| Array `[]`  | Ordered list                   | `arr[0]`       | `[1, 2, 3]`           |
| Map         | Key-value pairs (any key type) | `map.get(key)` | `new Map([['a', 1]])` |

## üîÅ 6. `for...in` vs `for...of`

| Loop       | Works on           | Iterates over             | Example                |
| ---------- | ------------------ | ------------------------- | ---------------------- |
| `for...in` | Objects            | **Keys** (property names) | `{a:1, b:2}`           |
| `for...of` | Arrays, Maps, Sets | **Values**                | `[1,2,3]`, `new Map()` |

```js
const arr = [10, 20, 30];
for (let i in arr) console.log(i);   // 0,1,2
for (let v of arr) console.log(v);   // 10,20,30
```

## üßÆ 7. `slice()` vs `splice()`

| Method                 | Mutates original? | Purpose                      |
| ---------------------- | ----------------- | ---------------------------- |
| `slice(start, end)`    | ‚ùå No              | Extracts part of an array    |
| `splice(start, count)` | ‚úÖ Yes             | Removes or replaces elements |

```js
const arr = [1, 2, 3, 4];
arr.slice(1, 3); // [2, 3]
arr.splice(1, 2); // removes [2,3], arr becomes [1,4]
```

## ‚ö° 8. `call()` vs `apply()` vs `bind()`

| Method    | Purpose                                        | Example                      |
| --------- | ---------------------------------------------- | ---------------------------- |
| `call()`  | Calls function with `this` and args separately | `fn.call(obj, a, b)`         |
| `apply()` | Calls function with `this` and args as array   | `fn.apply(obj, [a, b])`      |
| `bind()`  | Returns a new function with bound `this`       | `const bound = fn.bind(obj)` |

## üß± 9. `prototype` vs `__proto__`

| Concept      | Description                                                |
| ------------ | ---------------------------------------------------------- |
| `.prototype` | Property on *constructor functions* (used for inheritance) |
| `.__proto__` | Actual prototype of an *instance*                          |

```js
function Person() {}
const p = new Person();
console.log(p.__proto__ === Person.prototype); // true
```

## üåê 10. `async/await` vs `.then()`

| Style         | Description                   |
| ------------- | ----------------------------- |
| `.then()`     | Promise chaining syntax       |
| `async/await` | Modern async syntax (cleaner) |

```js
// then
fetch('/api').then(res => res.json()).then(data => console.log(data));

// async/await
const data = await (await fetch('/api')).json();
console.log(data);
```
