## ðŸ”¹ 1. What is a Promise?

## ðŸ”¹ 2. Why Promises?

Before Promises, async code was handled using **callbacks** â€” leading to two major problems:

1. **Inversion of Control** â€“ trusting another function to call your callback correctly.

## ðŸ”¹ 3. How Promises Fix It

### âœ… With Promises


### ðŸ§  Key Points:

* `.then()` attaches a callback to the Promise.
* `.catch()` handles errors.
* Each `.then()` returns a new Promise â†’ allows **Promise chaining**.
* **Promise guarantees**: callback runs **only once** when data is ready.

## ðŸ”¹ 7. Promise Chaining (Avoid Callback Hell)
