# ‚öôÔ∏è REACT REDUX ‚Äî Deep Concept Map

React Redux is the **official React binding for Redux**.
Redux itself is a **state management library** (not React-specific).
React Redux is what lets React components **talk to** Redux stores.

---

## üß± 1Ô∏è‚É£ Library Split

| Package           | Purpose                                                                                        | Works In                                       |
| ----------------- | ---------------------------------------------------------------------------------------------- | ---------------------------------------------- |
| **`redux`**       | The *core state container* ‚Äî defines store, reducer, dispatch, and actions.                    | Works with any JS app (Node, React, Vue, etc.) |
| **`react-redux`** | Connects React components to the Redux store. Provides React hooks and a `<Provider>` wrapper. | Works only in React apps                       |

So:

> `redux` = Brain üß† (state logic, data changes)
> `react-redux` = Bridge üåâ (connects that state to React UI)

---

## üß© 2Ô∏è‚É£ The Core Redux Concepts

Even before React comes in, Redux defines these fundamentals:

| Concept      | Description                                        | Example                                     |
| ------------ | -------------------------------------------------- | ------------------------------------------- |
| **Store**    | Central object holding all app state               | `const store = configureStore({ reducer })` |
| **Action**   | A plain object describing ‚Äúwhat happened‚Äù          | `{ type: 'ADD_USER', payload: data }`       |
| **Reducer**  | A pure function that changes state based on action | `(state, action) => newState`               |
| **Dispatch** | The function used to send an action to the store   | `store.dispatch({ type: 'ADD_USER' })`      |
| **Selector** | Function to extract data from state                | `(state) => state.user`                     |

Redux itself doesn‚Äôt know React ‚Äî it‚Äôs just JS logic.

---

## ‚öôÔ∏è 3Ô∏è‚É£ React-Redux Adds React Bindings

### It provides **two main things**:

1. **`<Provider store={store}>`**

   * Puts the Redux store into React‚Äôs *context system* so all components can access it.
   * Usually wraps your whole `<App />`.

   ```jsx
   import { Provider } from "react-redux";
   import { store } from "./store";

   createRoot(document.getElementById("root")).render(
     <Provider store={store}>
       <App />
     </Provider>
   );
   ```

   ‚ûú Without Provider, React components can‚Äôt reach the store.

---

2. **Hooks to use Redux inside components**

| Hook            | From Package  | Purpose                                  |
| --------------- | ------------- | ---------------------------------------- |
| `useSelector()` | `react-redux` | Read data from store                     |
| `useDispatch()` | `react-redux` | Send actions to the store                |
| `useStore()`    | `react-redux` | Direct access to the store object (rare) |

Example:

```jsx
import { useSelector, useDispatch } from "react-redux";
import { addUser } from "./userSlice";

function Profile() {
  const user = useSelector((state) => state.user);
  const dispatch = useDispatch();

  return (
    <div>
      <p>{user?.name}</p>
      <button onClick={() => dispatch(addUser({ name: "John" }))}>
        Add User
      </button>
    </div>
  );
}
```

## üèóÔ∏è 4Ô∏è‚É£ How It Works Behind the Scenes

### Step-by-step flow:

```
React Component ‚Üí dispatch(action) 
‚Üí Redux Store ‚Üí runs reducer ‚Üí updates state 
‚Üí Provider notifies components ‚Üí React re-renders those components
```

Detailed:

1. A component dispatches an action via `useDispatch()`.
2. Redux sends that action to the reducer(s).
3. Reducers compute a new state.
4. React-Redux Provider triggers a context update.
5. Components that used `useSelector()` see the new data and re-render.

---

## üßÆ 5Ô∏è‚É£ Redux Toolkit (RTK) ‚Äî Simplified Layer on Top

Modern Redux apps almost always use **Redux Toolkit** (`@reduxjs/toolkit`) ‚Äî which is built by the same team.

It gives you helper functions:

* `createSlice()` ‚Üí builds actions + reducers automatically
* `configureStore()` ‚Üí sets up store + middleware easily

Example:

```js
import { createSlice, configureStore } from "@reduxjs/toolkit";

const userSlice = createSlice({
  name: "user",
  initialState: null,
  reducers: {
    addUser: (state, action) => action.payload,
    removeUser: () => null,
  },
});

export const { addUser, removeUser } = userSlice.actions;
export const store = configureStore({ reducer: { user: userSlice.reducer } });
```

Then in your React app:

```jsx
<Provider store={store}>
  <App />
</Provider>
```

## üîÑ 7Ô∏è‚É£ Client vs Server Context
